# voc-report — 고객 피드백 수집·분석 리포트

## 언제 쓰나
댓글·DM·구매 후기 등 고객 반응을 모아 패턴을 찾고 다음 콘텐츠·상품에 반영할 때

## 자동 로드할 것
- @../../context/customer-segments.md (피드백 출처 세그먼트)
- @../../rules/고객응대원칙.md

---

## 입력 (Input)

| 항목 | 필수 여부 | 설명 |
|:-----|:---------|:-----|
| 피드백 원문 | 필수 | 댓글·DM·후기 텍스트 붙여넣기 |
| 수집 기간 | 필수 | 언제~언제 수집한 데이터인지 |
| 피드백 출처 | 권장 | 댓글 / DM / 구매 후기 구분 |

> 피드백 원문 없으면 분석 불가 — Zero-Inference 원칙 적용

---

## 3-Phase 파이프라인

### Phase 1. 분류
- 긍정 / 부정 / 요청(개선) / 질문으로 4분류
- 세그먼트별 피드백 패턴 식별

### Phase 2. 인사이트 추출
- 반복 키워드·패턴 Top 3 추출
- 콘텐츠 개선 시그널 (CCO 전달)
- 상품 개선 시그널 (CRO 내부 반영)
- 이탈 위험 시그널 (CXO 전달)

### Phase 3. 액션 아이템 + 납품
- 담당자·기한 포함 액션 최대 3개
- 마케팅 사업부 연계 필요 시 명시
