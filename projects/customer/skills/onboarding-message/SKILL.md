# onboarding-message — 구매자 온보딩 DM 시퀀스

## 언제 쓰나
전자책 구매자가 발생했을 때, 구매 직후부터 D+7까지 발송할 DM 시퀀스가 필요할 때

## 자동 로드할 것
- @../../context/customer-segments.md (고객 세그먼트 정의)
- @../../context/journey-map.md (구매 후 여정)
- @../../rules/고객응대원칙.md (응대 원칙·금지행동)

---

## 입력 (Input)

| 항목 | 필수 여부 | 설명 |
|:-----|:---------|:-----|
| 구매 상품명 | 필수 | 어떤 전자책인지 |
| 구매자 특성 | 권장 | 직장인·부업러 등 세그먼트 |
| 상품 핵심 가치 | 권장 | 이 전자책으로 얻는 Before→After |

---

## 3-Phase 파이프라인

### Phase 1. 여정 설계
- 구매자 세그먼트 확인
- D0~D+7 접점 4개 타이밍 설정
- 각 접점의 감정 목표 설정 (확신 → 실행 → 관계 → 연결)

### Phase 2. 메시지 작성
4개 메시지 순차 작성:
- **메시지 1 (D0, 즉시)**: 감사 + 사용 가이드 — 구매 확신
- **메시지 2 (D+1)**: 활용 팁 1개 — 첫 실행 유도
- **메시지 3 (D+3)**: 피드백 요청 — 관계 형성 + VOC 수집
- **메시지 4 (D+7)**: 다음 단계 자연스러운 연결 (강요 없이)

### Phase 3. 검수 후 납품
- 검수 기준 자동 체크
- 소셜비즈 입력 가이드 포함

---

## 산출물 형식

```
[온보딩 DM 시퀀스 — {상품명}]

메시지 1 (구매 즉시)
발송 트리거: 결제 완료
내용: ...
CTA: ...

메시지 2 (D+1)
발송 트리거: 구매 24시간 후
내용: ...
CTA: ...

메시지 3 (D+3 — 피드백)
발송 트리거: 구매 72시간 후
내용: ...
CTA: (피드백 요청)

메시지 4 (D+7 — 연결)
발송 트리거: 구매 7일 후
내용: ...
CTA: (다음 단계 안내)

[소셜비즈 입력 가이드]
트리거 설정 위치: ...
각 메시지 딜레이 설정: ...
```

## 검수 기준

| ID | 기준 | 합격 조건 | 가중치 |
|:---|:-----|:----------|:-------|
| O1 | 재구매 압박 없음 | D+3 이전 상품 언급 없음 | 필수 |
| O2 | 메시지당 CTA 1개 | 행동 요청 정확히 1개 | 필수 |
| O3 | 따뜻한 톤 | 공지·푸시 톤 없음 | 필수 |
| O4 | 4개 완성 | D0·D+1·D+3·D+7 모두 작성 | 필수 |
| O5 | 소셜비즈 가이드 | 입력 위치 명시 | 필수 |
