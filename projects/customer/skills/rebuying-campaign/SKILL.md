# rebuying-campaign — 재구매·업셀 캠페인 메시지

## 언제 쓰나
신규 전자책 출시 또는 기존 구매자에게 다음 상품을 제안할 때

## 자동 로드할 것
- @../../context/customer-segments.md (타겟 세그먼트)
- @../../rules/고객응대원칙.md (금지 행동 포함)

---

## 입력 (Input)

| 항목 | 필수 여부 | 설명 |
|:-----|:---------|:-----|
| 제안 상품명 | 필수 | 어떤 전자책·프롬프트인지 |
| 타겟 세그먼트 | 필수 | 1회 구매자 / 재구매자 중 택1 |
| 이전 구매 상품 | 권장 | 연결 고리 만들기 위해 |
| 캠페인 시기 | 권장 | 출시일·특정 이벤트 |

---

## 3-Phase 파이프라인

### Phase 1. 세그먼트 × 상품 연결
- 타겟 세그먼트의 현재 문제 정의
- 이전 구매 → 다음 문제 → 신규 상품 연결 논리 구성

### Phase 2. 메시지 3버전 작성
1. **스토리형**: 이전 상품 후기 → 다음 문제 → 자연스러운 제안
2. **가치형**: 이 상품으로 얻는 Before→After 명확히
3. **긴급형**: 출시 초기·한정 수량 (희소성 조작 없이 사실만)

### Phase 3. 검수 후 납품
- 검수 기준 자동 체크
- 발송 타이밍·빈도 가이드 포함

```
[재구매 캠페인 — {상품명}]

메시지 A (스토리형): ...
CTA: ...

메시지 B (가치형): ...
CTA: ...

메시지 C (긴급형, 선택): ...
CTA: ...

[운영 가이드]
발송 빈도: 월 최대 2회
추천 발송 시간: ...
```

## 검수 기준

| ID | 기준 | 합격 조건 | 가중치 |
|:---|:-----|:----------|:-------|
| B1 | 가치 우선 | 고객 문제 언급 후 상품 제안 | 필수 |
| B2 | 할인 표현 없음 | "최저가·세일·할인" 없음 | 필수 |
| B3 | 성과 보장 없음 | 효과 단정 표현 없음 | 필수 |
| B4 | CTA 단일 | 메시지당 1개 | 필수 |
| B5 | 빈도 가이드 | 월 최대 2회 명시 | 필수 |
