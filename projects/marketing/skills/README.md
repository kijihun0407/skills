# skills/ — 재사용 스킬 정의

> 반복 작업은 스킬로 만들어 즉시 호출합니다.
> 각 스킬은 하위 폴더 + `SKILL.md`로 구성합니다: `skills/{스킬명}/SKILL.md`

---

## 폴더 구조

```
skills/
├── README.md          ← 지금 여기
└── {스킬명}/
    └── SKILL.md       ← 스킬 정의 파일
```

---

## SKILL.md 형식 (필수 섹션)

```markdown
---
name: 스킬명
description: 한 줄 설명 (언제 이 스킬을 쓰는가)
trigger: 사용자가 뭐라고 말할 때 호출하는가
---

# [스킬명]

## 목적
이 스킬이 달성하는 결과물 (Before → After 명확히)

## 입력 (Input)
| 항목 | 필수 여부 | 설명 |
|:-----|:---------|:-----|
| 주제 | 필수 | ... |

## 실행 단계 (Pipeline)
1. Phase 1: [단계 설명]
2. Phase 2: [단계 설명]
3. Phase 3: [최종 납품]

## 산출물 형식
[결과물이 어떤 형태로 나와야 하는가]

## 검수 기준
| ID | 기준 | 합격 조건 | 가중치 |
|:---|:-----|:----------|:-------|
| Q1 | ... | ... | 필수 |
| Q2 | ... | ... | 권장 |

## 학습 기록
> CEO 피드백 발생 시 여기에 추가됩니다.
```
