# weekly-report — 주간 성과 데이터 출력

## 목적
원시 데이터 입력 → 인사이트 + 다음 주 액션 아이템 납품
Before: 숫자 나열 / After: 판단 가능한 인사이트 + 실행 계획

## 입력 (Input)

| 항목 | 필수 여부 | 설명 |
|:-----|:---------|:-----|
| 팔로워 수 (주초/주말) | 필수 | 순증감 계산용 |
| 릴스별 조회수·저장수·공유수 | 필수 | 게시물 단위 |
| 캐러셀별 저장수·도달수 | 필수 | 게시물 단위 |
| DM 수신 수 | 필수 | 자동화 포함 총 건수 |
| 전자책 결제 수 / 스폰서십 문의 수 | 권장 | 수익 전환 지표 |
| 지난 주 액션 아이템 | 권장 | 실행 여부 검토용 |

> 데이터 없는 항목은 분석에서 제외. 추측 금지.

## 실행 파이프라인

### Phase 1 — 데이터 정리
- 입력값 표로 정리
- 이상값(급등·급락) 플래그

### Phase 2 — 인사이트 추출
- 이번 주 Best 콘텐츠 1개 선정 + 이유
- Worst 콘텐츠 1개 선정 + 원인 가설
- 퍼널 병목 구간 진단 (조회 → 저장 → DM → 전환 중 어디서 막히는가)

### Phase 3 — 액션 아이템 + 납품
- 다음 주 실행 항목 최대 3개 (담당 에이전트·기한 포함)
- outputs/ 자동 저장: `outputs/{연도}/{월}/{일}/[분석]_주간성과_N주차.md`

## 검수 기준

| ID | 기준 | 합격 조건 | 가중치 |
|:---|:-----|:----------|:-------|
| W1 | Zero-Inference | 입력 없는 지표 추측 없음 | 필수 |
| W2 | Best/Worst 선정 | 각 1개, 이유 명시 | 필수 |
| W3 | 퍼널 병목 진단 | 막힌 구간 1개 이상 특정 | 필수 |
| W4 | 액션 아이템 | 담당자·기한 포함, 최대 3개 | 필수 |
| W5 | 비교 기준 | 지난 주 수치와 비교 (없으면 생략 명시) | 권장 |
