# context/ — 배경지식·톤앤매너·채널 규칙

> 프로젝트의 맥락 정보를 저장합니다.
> CLAUDE.md에서 `@context/파일명.md`로 필요한 시점에 임포트합니다.

---

## 권장 파일 구성

| 파일 | 내용 | 임포트 시점 |
|:-----|:-----|:------------|
| `tone.md` | 톤앤매너, 언어 스타일, 금지 표현 | 콘텐츠 작업 시 |
| `audience.md` | 타겟 고객 프로파일, 고객 여정 | 기획·마케팅 작업 시 |
| `brand.md` | 브랜드 아이덴티티, 포지셔닝 | 대외 커뮤니케이션 시 |
| `channels.md` | 채널별 규칙 (인스타/유튜브/등) | 채널 운영 작업 시 |
| `competitors.md` | 벤치마크·경쟁사 분석 | 전략·콘텐츠 기획 시 |

---

## 파일 형식 (예시: tone.md)

```markdown
# 톤앤매너

## 기본 원칙
1. [원칙 1]
2. [원칙 2]

## 금지 표현
- [쓰면 안 되는 말/패턴]

## 권장 표현
- [대신 쓸 말/패턴]

## 예시
- ❌ 잘못된 예: "...하면 어떨까요?"
- ✅ 올바른 예: "...하세요."
```
