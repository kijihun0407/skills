# CLAUDE.md — 마케팅 사업부 운영 시스템

> 한 줄 미션: AI 실용 콘텐츠로 인스타 잠재 고객을 확보하고 광고 및 전자책 수익으로 연결한다.

---

## 이 시스템은 무엇인가

- **미션**: AI 실용 콘텐츠로 인스타그램 잠재 고객을 확보하고, 브랜드 협업 광고 및 전자책 판매 수익으로 연결한다
- **핵심 업무**: 주 4회 릴스 발행 (트래픽 유입) + 주 2회 캐러셀 발행 (전문성 입증) + 잠재 고객 데이터(리드) 수집
- **대상**: AI를 업무나 일상에 활용하고 싶은 직장인 및 일반인
- **핵심 가치**: 검증된 정보만, 실제로 써먹을 수 있게, 가장 빠르게

---

## 아키텍처

```
new-project/
├── CLAUDE.md        ← 지금 여기 (진입점 — Claude Code 자동 로드)
├── .claude/
│   ├── settings.json  ← 권한·훅·환경변수 (하네스 설정)

├── agents/          ← AI 에이전트 페르소나 정의
├── skills/          ← 재사용 가능한 스킬 (SKILL.md 형식)
├── context/         ← 배경지식·톤앤매너·채널 규칙
└── rules/           ← 업무 규칙 (CLAUDE.md에서 @import)
```

## 책임 범위

### 이 사업부가 책임지는 것

| 영역 | 세부 내용 | 주기 |
|:-----|:----------|:-----|
| **콘텐츠 생산** | 릴스 (트래픽 유입) | 주 4회 |
| **콘텐츠 생산** | 캐러셀 (전문성 입증) | 주 2회 |
| **데이터·전환** | 주간 성과 분석 | 주 1회 |
| **데이터·전환** | DM/댓글 자동화(소셜비즈)로 리드 수집 | 상시 |

### 이 사업부가 책임지지 않는 것
- 유튜브·블로그 등 인스타 외 채널 운영 (추후 확장 시 별도 정의)
- 오프라인 강의·컨설팅

## 수익 구조

| 수익원 | 형태 | 전환 경로 |
|:-------|:-----|:----------|
| 브랜드 스폰서십 | 광고 협업·PPL | 팔로워 신뢰 → 브랜드 DM 문의 |
| 전자책 | 실전 지식 상품 | 콘텐츠 → 프로필 링크 → 결제 |

---

## 고객 여정 (콘텐츠 → 결제)

```
[릴스/카드뉴스] → 팔로우 → 프로필 탐색 → 무료자료 → 전자책 결제
                                                    ↘ 브랜드 DM 문의
```

| 단계 | 터치포인트 | 목표 행동 |
|:----:|:-----------|:----------|
| 1 | 릴스·카드뉴스 | 조회·저장·공유 |
| 2 | 팔로우·프로필 | 신뢰 형성 |
| 3 | 무료자료·링크 | 이메일·DM 확보 |
| 4 | 전자책·협업 DM | 결제·문의 |

> 모든 콘텐츠 작업 시 "이 콘텐츠는 어느 단계를 위한 것인가"를 먼저 정의한다.

| 에이전트 | 역할 | 소집 조건 |

| **CDO** | 주간 성과 데이터 분석·KPI 추적·알고리즘 대응·개선 전략 총괄 | 데이터 분석·리포트·전략 수정 시 |

## 금지 행동

- 허위·과장된 성과 약속 ("이걸 하면 반드시 수익 난다" 류)
- 어려운 전문 용어 단독 사용 — 반드시 비유·쉬운 설명 병기
- 경쟁 크리에이터 비방 또는 콘텐츠 완전 카피
- 데이터 없이 수치 추측 후 사실처럼 제시
- 스폰서십 협업 시 제품 미사용·미검증 상태로 추천
