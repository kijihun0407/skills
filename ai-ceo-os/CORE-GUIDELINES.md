# PART 2. AI OS 핵심 철학

## AI OS vs AI 도구
| 구분 | AI 도구 (ChatGPT) | AI OS (ai-ceo-os) |
|:---|:---|:---|
| 기억 | 매번 리셋 | @include로 영구 기억 |
| 실행 | 매번 20줄 | 스킬 1줄 / 커맨드 1줄 |
| 규칙 | 매번 말해야 함 | rules/로 자동 적용 |
| 판단 | 1명 관점 | 8명이 각자 관점으로 |
| 성장 | 안 함 | lessons + 주간 회고로 매주 진화 |

## 가치 사다리 (자동화 엔진)
```
[1단계] 릴스/유튜브 무료 콘텐츠 → AI가 대본 자동 생산
[2단계] 무료 PDF/체크리스트 → 댓글 키워드 → DM 자동 전달
[3단계] 전자책/강의 → 콘텐츠 → 커뮤니티 → 전환 퍼널
[4단계] 프리미엄 컨설팅 → 팬덤이 자연스럽게 이동
```

## 수익화 우선순위
트래픽 → 시장 테스트 → 잠재 고객 확보 → 판매 → 상품 개발

# PART 4. 콘텐츠 제작 원칙

## 콘텐츠 품질 기준 (릴스 대본)
| ID | 기준 | 합격 조건 |
|:---|:---|:---|
| Q1 | 첫 문장 후킹 | 3초 내 시선 잡는 문장 (질문형/수치형/반전형) |
| Q2 | 글자 수 | 180~250자 |
| Q3 | CTA | 정확히 1개 |
| Q4 | 인사말 없음 | "안녕하세요/오늘은/여러분" 금지 |
| Q5 | 전문용어 풀어쓰기 | "쉽게 말하면..." 포함 |
| Q6 | 구체적 숫자/사례 | 추상적 표현 금지 |

## V.I.R.A.L 프레임워크
| 알파벳 | 의미 | 적용 |
|:---|:---|:---|
| **V** | Viral Recycling | 잘 된 후킹 패턴 스킬에 저장해 재활용 |
| **I** | Insight Delivery | 모든 줄이 인사이트 (filler 금지) |
| **R** | Reverse Scripting | CTA 먼저 결정하고 역순으로 대본 설계 |
| **A** | Attention Hooks | 후킹 요소 최소 3개 |
| **L** | Loop Analytics | 성과 데이터 → 다음 콘텐츠에 반영 |

## 원소스 멀티유즈
```
릴스 대본 1개 →
  ├── 릴스 발행
  ├── 캐러셀(카드뉴스)로 재가공
  ├── 블로그/스레드 텍스트로 변환
  └── 유튜브 쇼츠로 확장
```

## 발행 스케줄
- 릴스: 주 3회 (월·수·금)
- 카드뉴스: 주 1~2회
- 해시태그: 5개 이내, CTA: 게시물당 1개

## 시리즈형 릴스 — 10-Beat 플로우
| Beat | 역할 | 패턴 |
|:---:|:---|:---|
| 1 | 시리즈 인트로 | 전편 동일 반복 |
| 3 | 공감 훅 | "~해본 적 있죠?" 패턴 |
| 5 | 원인 설명 | "이게 왜 그러냐면" 고정 |
| 6 | 솔루션+비유 | "쉽게 말하면 ~" 비유 필수 |
| 8 | 꿀팁 | "꿀팁은요," 고정 시작 |
| 9 | 임팩트 수치 | Before→After 숫자 |
| 10 | 권위+CTA | 권위 출처 + 명사 키워드 CTA |

# PART 6. 자동화 3단계 시스템

## 스킬 → 커맨드 → 크론
| 단계 | 이름 | 누가 시작 | 목표 수량 |
|:---|:---|:---|:---|
| 1 | **스킬 (skills/)** | 내가 시킴 | 34개+ |
| 2 | **커맨드 (commands/)** | 내가 시킴 | 30개+ |
| 3 | **크론 (cron)** | 자동 실행 | 26개+ |

## 스킬 역설계 3단계 (핵심 제작법)
```
① 대화: 평소처럼 AI에게 일을 시킨다 → 결과물 90점까지 올린다
② 스킬화: "이 작업을 스킬로 만들어줘" → AI가 패턴 역설계
③ 커맨드화: 여러 스킬 반복 시 → "커맨드로 묶어줘"
```

## RAW → YOURS 원칙
AI 냄새 감지법: "결론적으로", "다양한", "~라고 할 수 있습니다" → 내 말투로 교체

# PART 8. 바이브코딩 원칙 (개발 사업부)

## 핵심 공식
**바이브코딩 = 내 기획력 × AI 실행력**

## 생산자 3원칙 (WHAT)
1. **문제 먼저** — "누가 돈 내고 쓸 문제인가?"
2. **가치 먼저** — "사용자가 다시 올 이유가 있는가?"
3. **수익 먼저** — "좋아요 100개보다 결제 1건"

## DCR 원칙 (HOW)
- **D**efine: PLAN.md 기획서 먼저 (없으면 시작 금지)
- **C**onverse: 쪼개서 대화 (한방 프롬프트 금지)
- **R**efine: 에러 복붙해서 AI한테 물어보기

## 개발 10습관
1. 기획서(PLAN.md) 없으면 코드 시작 금지
2. "이게 진짜 문제인가?" 되묻기
3. GitHub 5분 검색 먼저
4. **환경 확인 먼저, 추정 금지** (★ 40분 장애 교훈)
5. "쉽게 말하면?" 되묻기
6. **10분마다 커밋** (롤백 안전선)
7. **테스트 안 했으면 완료 아님** (★ HTTP 200 사고 교훈)
8. 선택지 2개 비교 → 근거 결정
9. 실패 기록 → lessons.md
10. 크로스 도메인 체크

## 에이나우 고정 스택 6개
Next.js / Tailwind / Supabase / Claude API / Vercel / 토스페이먼츠

## 보안 필수 3개
