# 하네스 엔지니어링 (Harness Engineering)

> 말을 탈 때 고삐·안장·등자가 필요하듯, AI를 일 시키려면 "장치 세트"가 필요하다.
> 이 5개가 꽉 물려 있어야 AI가 내 사업부에서 "자기 일"처럼 일한다.
> 하나라도 빠지면 말에서 떨어진다.

---

## 하네스 5종 세트

```
projects/xxx/
├── CLAUDE.md              ← ① 사업부 헌법
├── agents/                ← ② 전담 직원 (본사 C-Suite에서 선발)
│   └── README.md
├── skills/                ← ③ 업무 매뉴얼 (SOP)
│   └── [스킬명].md
├── context/               ← ④ 참고 자료 (채널 규칙, 데이터)
│   └── [규칙명].md
└── rules/                 ← ⑤ 자동 적용 사규 (금지/품질)
    └── [규칙명].md
```

## 각 하네스의 역할

| # | 하네스 | 역할 | 회사 비유 | 없으면? |
|:---|:---|:---|:---|:---|
| ① | **CLAUDE.md** | 이 사업부의 미션·KPI·담당자 | 부서 오리엔테이션 문서 | 팀이 방향 없이 떠돔 |
| ② | **agents/** | 담당 임원 2~3명 + 본사 @참조 | 부서 조직도 | "누가 할 일인데요?" |
| ③ | **skills/** | 반복 작업 절차서 (Phase별) | 부서 SOP 매뉴얼 | 매번 처음부터 설명 |
| ④ | **context/** | 이 팀만 아는 세부 정보 | 부서 업무 노트 | 결과가 범용적 |
| ⑤ | **rules/** | 금지·품질 룰 자동 적용 | 부서 사규 | 사고 반복 |

## 왜 본사와 분리하는가

### 본사 CLAUDE.md의 한계

본사 CLAUDE.md는 **회사 전체 헌법**이다. 인스타 채널 규칙 같은 세부 사항이 들어가면 파일이 비대해져서 AI가 중요한 걸 놓친다.

→ 본사는 요약만 두고, **세부 맥락은 사업부로 내려보낸다.**

### 같은 질문, 다른 결과

```
[본사 루트에서]
"이번 주 릴스 3개 기획해줘"
→ 범용 답변 (AI 활용 팁, 트렌드 정리...)
→ 내 사업/톤/채널 반영 부족

[마케팅 사업부 폴더에서]
"이번 주 릴스 3개 기획해줘"  ← 완전 똑같은 질문
→ CCO + CMO 자동 소집
→ channel-rules.md + 금지행동.md 자동 로드
→ 내 업종·톤·CTA·발행 시간까지 맞춘 답
```

## 쪼개기 원칙 3가지

### 원칙 1. 채널이 아니라 업무 카테고리로 쪼갠다

| 비교 | ❌ 채널로 쪼개기 | ✅ 업무로 쪼개기 |
|:---|:---|:---|
| 구조 | insta/ · youtube/ · blog/ | marketing/ · dev/ · operations/ · customer/ |
| 문제 | 같은 스킬을 3개 폴더에 복붙 | 업무 로직 재사용, 협업 명확 |

### 원칙 2. 처음엔 2~3개로 시작한다

한 번에 6개 사업부 만들면 다 무너진다. 2개만 먼저 완성 → 체득 → 추가.

### 원칙 3. 하나 잘 만들고 복제한다

첫 사업부가 템플릿이 된다:
- 첫 번째: 30~40분
- 두 번째: 10분 (복제 + 내용 치환)
- 세 번째: 5분

## @참조 패턴 (agents/ 연결법)

본사 임원을 복사하지 않고 **경로로 연결**한다. 본사에서 CCO 업데이트하면 모든 사업부에 자동 반영.

```markdown
### 담당: CCO
- 본사 경로: @../../agents/cco/README.md
- 이 사업부에서의 추가 역할: ...
```

## 연결 구조 시각화

```
┌────────────────────────────────┐
│   본사 CLAUDE.md (전사 헌법)    │
└──────────────┬─────────────────┘
               │ (항상 상위로 상속)
    ┌──────────┼──────────┐
    ▼          ▼          ▼
marketing/ customer/   dev/
CLAUDE.md  CLAUDE.md  CLAUDE.md
+ CCO,CMO  + CXO,CCO  + CTO
+ 콘텐츠   + CS스킬   + 개발스킬
```

cd 명령으로 어느 사업부에 들어가느냐 = 누구를 호출하느냐.
