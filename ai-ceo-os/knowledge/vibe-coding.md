# 바이브코딩 — 개념 + 원칙 + 방법론

> 바이브코딩 = 내 기획력 × AI 실행력
> 내 기획력이 0이면 0 × AI = 0. 기획력이 10이면 10 × AI(10) = 100.
> 여러분이 키워야 할 건 AI가 아니라 기획력이다.

---

## 바이브코딩 정의

"코딩을 AI에게 '말'로 시키는 것."

| 항목 | 기존 코딩 | 바이브코딩 |
|:---|:---|:---|
| **입력** | 키보드 타이핑 | 말 / 기획서 |
| **핵심 능력** | 코드를 쓰는 능력 | AI에게 정확히 시키는 능력 |
| **한계** | 내 손 속도 | 내 기획력 |

### 왕초보 3대 오해 바로잡기

❌ **오해 #1: "바이브코딩 = 노코드"**
→ 노코드는 템플릿 조립. 바이브코딩은 진짜 코드를 AI가 생성. 노코드는 템플릿 벗어나면 못 만들지만, 바이브코딩은 아무거나 만들 수 있다.

❌ **오해 #2: "AI가 다 해주니까 저는 놀면 되나요?"**
→ 기획 안 된 입력 = 제네릭 쓰레기 출력. AI는 사장님이 시키는 일만 한다.

❌ **오해 #3: "어차피 저는 안 될 것 같아요"**
→ 키워야 할 건 딱 두 가지: 기획력 + AI와 대화하는 법. 코드 문법은 AI가 안다.

## 생산자 3원칙 (WHAT — 무엇을 만들까)

### 원칙 1: 문제 먼저 (Problem First)
"누가 돈 내고 쓸 문제인가?"

- ✅ 문제: "고3 수능 D-30 스트레스 폭발" (학부모도 돈 냄, 학생도 돈 냄)
- ❌ 문제 아님: "그림 잘 그리고 싶다" (돈 낼 사람 없음, 취미)

### 원칙 2: 가치 먼저 (Value First)
"사용자가 다시 올 이유가 있는가?"

| 구분 | Case 1: 바이럴 | Case 2: 재방문 |
|:---|:---|:---|
| 1회 방문 | 1만 명 | 100명 |
| 재방문율 | 2% | **30%** |
| 광고비 | 500만원 | 0원 |

→ 광고 끊으면 0인 Case 1보다 광고 없이도 재방문하는 Case 2가 장기적으로 이긴다.

### 원칙 3: 수익 먼저 (Revenue First)
"좋아요 100개보다 결제 1건"

목표 = D+30까지 첫 결제 10만원 (Bronze 인증 기준)

## CEO 바이브코딩 5단계 (HOW)

| Step | 이름 | 한 줄 요약 |
|:---:|:---|:---|
| 1 | **CLAUDE.md 먼저** | AI에게 규칙부터 주면 일관성 보장 |
| 2 | **고정 스택** | 선택지 10개 비교 X, 검증된 6개만 |
| 3 | **말로 설명 (DCR)** | 타이핑 속도 < 말 속도 |
| 4 | **git push = 배포** | 수동 업로드 = 사고. 자동화 = 안전 |
| 5 | **써보며 고침** | 배포가 끝이 아니라 시작 |

## DCR 3원칙 (HOW — AI와 일하는 법)

### D — Define (정의)
"기획서 먼저. PLAN.md. AI한테 시키기 전에 '뭘 만들 건지' 정의."

- Before: "블로그 만들어줘" → 제네릭 결과
- After: PLAN.md → 1번에 거의 완성

### C — Converse (쪼개서 대화)
"코딩이 아니라 대화. 한 번에 완벽한 결과 기대 금지."

- Before: "디자인/반응형/DB/결제 한 번에" → 에러 폭발
- After: 5단계 순차 → 에러 거의 없음

### R — Refine (정제)
"에러도 AI한테 '이거 어떻게 고칠까?' 물어봐요. 에러 스크린샷 그대로 복붙도 OK."

**운전 비유**: 빨간불 만났다고 차 버리는 사람 없잖아요.

## 기획 도구 3종

| 도구 | 레벨 | 언제 | 비유 |
|:---|:---|:---|:---|
| **PLAN.md** | 프로젝트 전체 | 시작 시 1회 | 프로젝트 지도 |
| **Plan Mode** (`Shift+Tab`) | 작업 단위 | 코드 쓸 때마다 | 작업 레시피 |
| **INTERVIEW.md** | 기획 가속 | 백지 공포증일 때 | PLAN.md 치트키 |

### PLAN.md 11섹션 구조
§1 한 줄 정의 / §2 타겟 / §3 해결(Before/After) / §4 핵심 기능 3개
§5 차별점 / §6 성공 기준 / §7 수익화
§8 생산자 3원칙 자가 답변 / §9 10습관 자가 체크
§10 기술 스택 / §11 변경 이력

### Plan Mode 사용법
- 단축키: `Shift + Tab`
- 또는 "Plan Mode로 진입해" / "/plan"
- 나오기: "이 계획대로 진행해줘"
- 효과: 결과물 품질 3배 상승

### INTERVIEW.md 치트키 프롬프트
```
너는 내 프로젝트 기획 확정을 도와주는 PM이야.
규칙: 한 번에 질문 1개씩, 10개까지, 끝나면 PLAN.md 자동 채워줌
질문: 타겟 / 문제 / 현 해결책 / 단점 / 차별점 /
핵심 기능 3개 / 가격 / 수익화 / 3개월 후 / 실패 정의
시작해줘. Q1부터.
```
→ 10분이면 PLAN.md 완성

### 완전 워크플로우
```
프로젝트 시작 (1회):
  INTERVIEW.md 프롬프트 → AI가 10개 질문 → 답변 10개 → PLAN.md 자동 완성

코드 쓸 때마다 (매번):
  Shift + Tab = Plan Mode → "이 기능 만들 계획 세워줘" → 계획 확인 후 진행
```
