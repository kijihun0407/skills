## 시드 (초기 학습)

### 콘텐츠
- **[2026-04-10] RULE** 릴스 대본은 반드시 30~60초 분량으로 제한. 그 이상은 이탈률 급증.
- **[2026-04-10] RULE** 카드뉴스 첫 장에는 숫자 포함 제목이 저장률 높음. (예: "AI 툴 TOP 5")
- **[2026-04-10] RULE** CTA는 1개만. 팔로우 + 저장 + 댓글 동시 요청 시 전환율 하락.

### 정보 품질
- **[2026-04-10] RULE** AI 기능/수치는 반드시 공식 발표 또는 직접 테스트 후 게시. 추측 금지.
- **[2026-04-10] RULE** "모든 사람이 모른다", "80%가 못한다" 식의 근거 없는 통계 사용 금지.

### 운영
- **[2026-04-10] RULE** 직장 병행 → 콘텐츠 제작은 배치로. 주말에 3~5개 미리 제작, 평일 예약 게시.

### AI 자동화 설계

- **[2026-04-25] MISTAKE** 자동화 파이프라인 설명 시 AI 아바타(HeyGen)를 캐러셀과 혼동해서 언급. AI 아바타는 **릴스 전용**이고, 캐러셀은 **Canva Bulk Create(CSV)** 방식. 두 포맷의 도구를 절대 혼동하지 않는다.
  - 릴스 → HeyGen(AI 아바타) → 캡컷 후편집
  - 캐러셀 → Claude(carousel-script) → CSV → Canva Bulk Create

### 인스타그램 계정명 / 아이디 확인

- **[2026-04-25] MISTAKE** 인스타그램 계정 가용성을 curl HTTP 상태코드로 확인 → 완전히 잘못된 방법. 인스타그램은 존재하지 않는 계정도 200을 반환하므로 코드로 판별 불가.
  - **올바른 방법**: "인스타그램 앱에서 직접 아이디 입력해서 확인해 주세요"라고 안내
  - **금지**: curl/HTTP로 인스타 계정 가용성 체크 절대 금지

- **[2026-04-25] MISTAKE** 계정명 확인 전에 파일 전체를 업데이트함 → 사용 불가 계정으로 업데이트되는 사고 발생. 시간 낭비.
  - **올바른 순서**: 계정 가용성 확인(앱) → CEO 확정 → 파일 업데이트

### Apify 활용 범위 인식

- **[2026-04-25] MISTAKE** 인스타그램 URL 공유받고 "로그인 필요, 접근 불가"라고 응답 → 완전히 잘못된 반응. Apify가 이미 설치된 상황이므로 즉시 스크래핑해야 했음.
  - **올바른 방법**: 인스타그램 URL 받으면 → Apify instagram-scraper로 즉시 스크래핑
  - **금지**: Apify 설치된 상황에서 "인스타그램 접근 불가"라고 응답하는 것
