## 3-레이어 아키텍처 (개발 사업부)

| 레이어 | 개발 적용 | 위치 |
|:---|:---|:---|
| **Layer 1: Directive** | 자동화 스크립트 SOP, 랜딩페이지 제작 SOP | `directives/` |
| **Layer 2: Orchestration** | CTO가 Directive 읽고 실행 도구 선택 | AI (김이사) |
| **Layer 3: Execution** | 인스타 통계 수집, 구글 시트 동기화 등 | `../../execution/` |

### 도구 우선 확인 원칙
새 스크립트 작성 전 `../../execution/`에 유사 도구가 있는지 먼저 확인.
없으면 `directives/`의 해당 SOP를 참조해 작성.

### Self-Annealing
- 스크립트 에러 발생 → 스택 트레이스 분석 → 수정 → 재테스트
- API 제한·타이밍 이슈 발견 → 해당 `directives/` SOP에 엣지케이스 추가
- 유료 API 호출이 필요한 수정은 CEO 확인 후 진행
