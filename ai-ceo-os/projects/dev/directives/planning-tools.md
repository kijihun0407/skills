# SOP: 기획 도구 3종 활용

> 기획이 빠지면 완성도 50% 이하로 떨어진다.
> PLAN.md → Plan Mode → INTERVIEW.md 순으로 사용.

---

## 기획 도구 3종 매트릭스

| 도구 | 레벨 | 언제 | 비유 |
|:---|:---|:---|:---|
| **PLAN.md** | 프로젝트 전체 | 시작 시 1회 | 프로젝트 지도 |
| **Plan Mode** (`Shift+Tab`) | 작업 단위 | 코드 쓸 때마다 | 작업 레시피 |
| **INTERVIEW.md** | 기획 가속 | 백지 공포증일 때 | PLAN.md 치트키 |

## PLAN.md 작성 SOP

### Phase 1: AI 자가 인터뷰
프롬프트 (prompts/01-interview-setup.md 참조):
```
너는 내 프로젝트 기획 확정을 도와주는 PM이야.
규칙: 한 번에 질문 1개씩, 10개까지
끝나면 PLAN.md 자동 채워줌

질문 목록:
Q1. 누구를 위한 서비스인가?
Q2. 그 사람이 겪는 가장 큰 문제는?
Q3. 지금 어떻게 해결하고 있나?
Q4. 현재 해결책의 단점은?
Q5. 내 서비스만의 차별점은?
Q6. 핵심 기능 3개는?
Q7. 무료 vs 유료, 가격은?
Q8. 수익화 경로는?
Q9. 3개월 후 모습은?
Q10. 실패 정의는?

시작해줘. Q1부터.
```

### Phase 2: PLAN.md 11섹션 완성
```
§1 한 줄 정의 — 서비스를 한 문장으로
§2 타겟 — 누구를 위해
§3 해결 — Before/After
§4 핵심 기능 3개
§5 차별점
§6 성공 기준
§7 수익화
§8 생산자 3원칙 자가 답변
§9 10습관 자가 체크
§10 기술 스택
§11 변경 이력
```

### Phase 3: 저장
- 파일명: `PLAN.md` (대문자, 프로젝트 루트)
- CLAUDE.md와 같은 위치에 저장

## Plan Mode 활용 SOP

### 언제 쓰나
- 새 기능 구현 시작 전
- 에러가 복잡해서 어디서 시작할지 모를 때
- "이게 맞는 방향인가?" 확신이 없을 때

### 사용법
```
1. Shift + Tab 입력
2. "이 기능 만들 계획 세워줘"
3. AI가 단계별 계획 제시
4. 수정 사항 있으면: "3번 바꿔, 7번 빼줘"
5. 확정되면: "이 계획대로 진행해줘"
```

### Before/After
```
Before (Plan Mode 없이):
"블로그 만들어줘" → 제네릭 결과 → 수정 반복 → 시간 낭비

After (Plan Mode 있이):
Shift+Tab → 계획 확인 → 수정 → 진행 → 결과물 품질 3배
```
