# Directive: Claude Code 실전 사용 가이드

> Layer 1. CTO가 참조. Claude Code를 효율적으로 활용하기 위한 실전 SOP.

## 기본 실행

```bash
claude        # Claude Code 시작
/setup        # 초기 셋업 (처음 한 번만)
```

그냥 한국어로 말하면 된다: "이번 주 릴스 주제 3개 기획해줘"

---

## 필수 명령어 5개

| 명령어 | 언제 | 설명 |
|:---|:---|:---|
| **/help** | 길을 잃었을 때 | 전체 도움말 |
| **/clear** | 대화가 꼬였을 때 | 처음부터 다시 (기억 초기화) |
| **/cost** | 비용 궁금할 때 | 지금까지 얼마 썼는지 |
| **/model** | 모델 바꿀 때 | Opus(똑똑+비쌈) ↔ Sonnet(빠름+저렴) |
| **/compact** | 대화 길어졌을 때 | 대화 압축해서 AI 성능 유지 |

---

## 모델 선택 전략 (비용 절반으로 줄이기)

| 모델 | 특징 | 언제 | 사용 비율 |
|:---|:---|:---|:---|
| **Sonnet** | 빠르고 저렴 | 평소 작업 — 대본, 기획, 답변 | 95% |
| **Opus** | 똑똑하지만 느리고 비쌈 | 전략 판단, 중요 문서, 복잡한 분석 | 5% |

> 평소엔 Sonnet, 중요할 때만 `/model`로 Opus 전환 → 비용 절반 절감

---

## 키보드 단축키 3개

| 단축키 | 하는 일 | 언제 |
|:---|:---|:---|
| **Esc** (1번) | AI 작업 일시정지 (맥락 유지) | AI가 엉뚱한 방향으로 갈 때 |
| **Esc Esc** (2번 빠르게) | 이전 상태로 되돌리기 | 실수했을 때 복원 |
| **Ctrl+C** | 강제 중단 | 완전히 멈추고 싶을 때 |

## 허용 요청 처리 원칙

Claude Code가 파일 수정/명령 실행 시 허용 여부를 물어본다:
- **y**: 허용 (이번 한 번)
- **n**: 거절

> 대부분 y. 경고 메시지 없으면 y. 경고가 있으면 내용 읽고 판단.

---

## 파일 참조 (@파일명)

```
@CLAUDE.md 이 파일 요약해줘
@경쟁사분석.txt 차별점 3개 찾아줘
@고객후기.txt 공통 불만사항 정리해줘
@매출데이터.csv 트렌드 분석해줘
```

> `@`만 붙이면 파일을 바로 읽는다. ChatGPT에서 복사-붙여넣기 하던 것, 여기선 한 글자.

---

## 이미지/스크린샷 분석

경쟁사 인스타 캡처 후 Claude Code에 드래그:
```
"이 릴스 썸네일을 분석해줘. 후킹 요소가 뭐야?"
```
→ 후킹 멘트, 텍스트, 시각적 요소 자동 분석

```

5. "경쟁 계정 @___를 분석해서 배울 점 3가지 알려줘"
```

## 에러 대응

| 에러 유형 | 증상 | 해결법 |
|:---|:---|:---|
| **API 키 미설정** | "API key not found" | API 키 재설정 |
| **npm 설치 실패** | "npm ERR!" 빨간 글씨 | `npm install` 재실행 |
| **/setup 중단** | 질문 도중 멈춤 | Claude Code 재시작 후 `/setup` 재실행 |

---

## execution/ 스크립트 작성 시 원칙

1. `../../execution/`에 유사 스크립트 먼저 확인
2. API 키는 `.env`에서만 로드 (하드코딩 절대 금지)
3. 에러 발생 → 스택 트레이스 분석 → 수정 → 재테스트 → directive 업데이트
4. 유료 API 수정 → CEO 확인 먼저
