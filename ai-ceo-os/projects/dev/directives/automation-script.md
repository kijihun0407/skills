# Directive: 자동화 스크립트 작성 SOP (개발 사업부)

> Layer 1. CTO가 실행 지휘. 결정론적 코드로 반복 업무를 자동화한다.

## 목표

반복 업무를 Python 스크립트로 자동화해 직장 병행 시간을 확보한다.

## 입력값

- 자동화할 업무 설명 (CEO 또는 다른 사업부에서 요청)
- 필요한 데이터 소스 (인스타그램 API, Google Sheets 등)
- 출력 형태 (파일, 클라우드, 터미널)

## 실행 순서

1. `../../execution/`에 유사 스크립트 있는지 먼저 확인
2. 있으면 재사용/수정. 없으면 새로 작성
3. `.env`에서 API 키 로드 (하드코딩 절대 금지)
4. 로컬 테스트 (프로덕션 직접 테스트 금지)
5. `sanitize_for_public.py`로 민감정보 체크
6. `../../execution/`에 저장 + 주석 추가

## 스크립트 작성 규칙

- 파일명: `{기능}_{대상}.py` (예: `instagram_stats_fetch.py`)
- 상단 docstring: 목적, 입력, 출력, 필요 env var
- 에러 핸들링: try/except + 명확한 에러 메시지
- API 키: `.env`에서만 로드 (`python-dotenv` 사용)
- 임시 파일: `.tmp/` 폴더에만 저장

## 엣지케이스

- API 레이트 리밋 → 배치 엔드포인트 검토 후 Directive 업데이트
- 유료 API 호출이 필요한 수정 → CEO 확인 먼저
- 스크립트 에러 → 스택 트레이스 분석 → 수정 → 재테스트 → SOP 업데이트
