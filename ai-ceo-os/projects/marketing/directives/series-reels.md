# SOP: 시리즈형 릴스 제작

> CEO가 시리즈 콘셉트와 1편(레퍼런스) 대본을 주면, 2편~N편까지 동일한 플로우·길이·인트로·톤으로 풀세트를 생산한다.
> **핵심 원칙: 1편이 기준이다. 1편 플로우에서 벗어나지 않는다.**

---

## 10-Beat 플로우 (시리즈 표준 구조)

| Beat | 역할 | 패턴 |
|:---:|:---|:---|
| 1 | **시리즈 인트로** | 전편 동일 반복 |
| 2 | **에피소드 인트로** | N일차 + 주제명만 변경 |
| 3 | **공감 훅** | "~해본 적 있죠?" / "~하셨죠?" 패턴 |
| 4 | **구체적 예시** | "~했더니 ~해요" 실제 상황 묘사 |
| 5 | **원인 설명** | "이게 왜 그러냐면 ~해서 그래요" 고정 |
| 6 | **솔루션 + 비유** | "~하면, 쉽게 말하면 ~" 비유 필수 |
| 7 | **작동 방식** | 실제 동작을 대화체로 묘사 |
| 8 | **꿀팁** | "꿀팁은요," 고정 시작어 |
| 9 | **임팩트 수치** | 구체적 Before→After 숫자 |
| 10 | **권위 포인트 + CTA** | 권위 출처 + 명사 키워드 CTA |

### 비트별 적용 규칙
- **Beat 1~2**: 전편 동일 구조. 날짜/주제만 교체
- **Beat 5**: "이게 왜 그러냐면" 고정 문구
- **Beat 6**: **매편 다른 비유 필수** — 비유가 시리즈의 차별화 포인트
- **Beat 8**: "꿀팁은요," 고정 시작
- **Beat 9**: 반드시 숫자 대비 (X분→Y분, N배 등)

## V.I.R.A.L 프레임워크

| 알파벳 | 의미 | 스킬 적용 방식 |
|:---|:---|:---|
| **V** | Viral Recycling — 잘 된 구조 재활용 | 조회수 1만 넘은 릴스 후킹 패턴을 스킬 프로세스에 저장 |
| **I** | Insight Delivery — 관점 전달 | 품질 기준에 "모든 줄이 인사이트 (filler 금지)" 항목 추가 |
| **R** | Reverse Scripting — 결론 먼저 설계 | 프로세스 1단계를 "CTA 먼저 확정"으로 설정 |
| **A** | Attention Hooks — 첫 3초 장치 | 품질 기준에 "후킹 요소 3개 이상" 항목 추가 |
| **L** | Loop Analytics — 성과 데이터 반영 | knowledge/에 인사이트 데이터 → AI가 잘 된 포맷 자동 기획 |

## 산출물 포맷 (편당 8개 섹션)

각 편은 아래 8개 섹션을 모두 포함:

1. **기본 정보** — 시리즈, 회차, 주제, 유형, 타겟, CTA 키워드, 다음 편 예고
2. **제목 후보 5개** — 순위별, 1~2위 볼드 처리
3. **후킹 텍스트** — 0~3초 자막 + 전환점 텍스트
4. **대본 (~35초)** — 10-Beat 플로우 적용
5. **캡션** — 시리즈명 + 시즌 진행 상황 + CTA 키워드
6. **썸네일 카피 5안** — 상단 고정 + 하단 5가지
7. **후킹 체크리스트** — 10-Beat 준수 여부 체크
8. **편집 노트** — 화면 캡처, B롤, 텍스트 오버레이 지시

## 시리즈 설계 원칙

### 비유 설계 (핵심 차별화)
- 매편 다른 비유 (같은 비유 반복 금지)
- 한 문장 비유: "쉽게 말하면 ~하는 거예요"
- 타겟이 아는 비유 사용 (코딩 비유 X → 일상 비유 O)

### 썸네일 넘버링 전략
- 상단: 시리즈 타이틀 (전편 고정)
- 하단: "왕초보 가이드 N일차" 넘버링 통일
- 최종편: "N일차 (완결)" 표시

### CTA 키워드 규칙
- 명사만 사용 (형용사 금지)
- 해당 편 주제와 직결
- 댓글 → DM → PDF → 부트캠프/커뮤니티 퍼널 연결

### 캡션 시즌 진행 상황
```
시즌N 진행 상황
1일차: 주제 ✅
2일차: 주제 ✅
N일차: 주제 (오늘)
(N+1)일차~: 매일 공개
```

## 제작 프로세스

### Phase 1: 시리즈 셋업 (1회만)
1. CEO가 시리즈 콘셉트 제공 (주제, 편수, 타겟)
2. CEO가 1편 대본 제공 (레퍼런스)
3. 1편 대본을 10-Beat로 분해 → 시리즈 플로우 확정
4. 전편 주제 리스트 + 순서 확정

### Phase 2: 에피소드 제작 (편당)
1. 해당 편 주제의 핵심 메시지 정리 (역순 대본법 — CTA 먼저)
2. 10-Beat 플로우에 맞춰 대본 작성
3. 비유 선정 (이전 편과 중복 안 되는 일상 비유)
4. 8개 섹션 완성 + 시리즈 폴더에 저장

### Phase 3: 품질 검증
- 10-Beat 플로우 준수 여부
- 비유 중복 체크 (이전 편과 겹치지 않는지)
- 30~40초 분량 확인
- 후킹 3개+ 확인
- CTA 명사 키워드 확인

| E7 | CTA 명사 키워드 | 형용사 아닌 명사 | 필수 |

| E11 | 권위 포인트 | 공식 문서/전문가 인용 | 권장 |
