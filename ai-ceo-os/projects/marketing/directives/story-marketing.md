# Directive: 스토리 마케팅 SOP

> Layer 1. 릴스/피드가 알고리즘용이라면, 스토리는 진성 팔로워와의 소통 창구다.

## 목표

스토리를 통해 팔로워와 신뢰를 쌓고, 무형 상품/광고로의 자연스러운 전환을 유도한다.

## 릴스 vs 스토리 차이

| 구분 | 릴스/피드 | 스토리 |
|:---|:---|:---|
| 노출 대상 | 알고리즘 → 불특정 다수 | 팔로워 → 진성 팬 |
| 역할 | 완성된 결과물 공간 | 과정과 일상을 보여주는 공간 |
| 보관 | 영구 | 24시간 후 사라짐 (부담 없음) |
| 소통 방향 | 일방향 | 양방향 (설문/질문/퀴즈) |

## 스토리 소통 전략 2가지

### 전략 1: 질문과 피드백 유도

- "여러분은 어떻게 생각하세요?" / "다음 주제 투표해주세요"
- 질문 스티커, 투표 스티커로 참여 장벽 낮추기
- 팔로워 반응 → DM이나 다음 스토리에서 언급하며 답변
- "어제 질문 주신 내용 답변드립니다" → 팔로워가 중요하게 여겨진다는 느낌

### 전략 2: 비하인드와 과정 공유

- "오늘 이런 작업 중입니다" / "기획하면서 고민했던 부분"
- 실패한 테스트 결과도 공유 → 진정성
- 완벽한 결과물보다 과정 속 진정성에 더 공감
- 의도적으로 불완벽함 표현 → 인간적 면모 → 신뢰

## 스토리 활용 패턴 (주간)

| 시점 | 스토리 내용 | 목적 |
|:---|:---|:---|
| 릴스 발행 당일 | 릴스 공유 + "이 영상 어떠세요?" 질문 | 인게이지먼트 부스팅 |
| 콘텐츠 기획 중 | "이번 주 어떤 주제가 더 궁금하세요?" 투표 | 수요 검증 |
| 상품 출시 전 | "이런 자료 만들면 사실 분?" 수요 조사 | 판매 전 검증 |
| 수시 | 과정 공유, AI 트렌드 단신, 팔로워 질문 답변 | 신뢰 구축 |

## 무형 상품 수요 조사 (스토리 활용)

```
Step 1: "이런 [주제] 자료 만들면 사실 분?" 설문 스티커
Step 2: 반응 50명+ → 제작 결정
Step 3: "드디어 완성! 댓글에 '링크'라고 적어주세요" 릴스 발행
Step 4: 소셜비즈 DM 자동화로 구매 링크 발송
```

## 금지 행동

- 스토리에서 과도한 직접 판매 (관계 형성 공간임)
- 하루 10개+ 스토리 (피로감 유발)
- 콘텐츠 없이 광고만 올리기
