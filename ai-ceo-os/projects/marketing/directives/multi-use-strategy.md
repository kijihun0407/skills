# Directive: 원소스 멀티유즈 SOP

> Layer 1. 하나의 콘텐츠를 여러 채널에 맞게 가공해 효율을 극대화한다.
> 핵심: 같은 영상을 그대로 올리는 것이 아니라, 각 채널에 맞게 새롭게 가공.

## 3가지 멀티유즈 전략

### 전략 1: 최신 정보 공유 (커뮤니티·스토리·스레드용)
```
X(트위터) 정보 수집
→ 네이버 카페 공유
→ 인스타 스토리
→ 단톡방 정보 공유
→ 스레드 공유
```

### 전략 2: 콘텐츠 기획 (메인 콘텐츠)
```
유튜브 레퍼런스 수집
→ 유튜브 제작 (긴 버전)
→ 릴스 제작 (30~40초 핵심)
→ 캐러셀 제작 (슬라이드화)
→ 스토리 공유
→ 카페 공유
→ 단톡방 공유
```

### 전략 3: 인스타그램 빠른 순환
```
릴스 제작
→ 캐러셀 제작 (같은 주제 슬라이드화)
→ 스레드 업로드 (텍스트 버전)
```

## 자동 멀티배포 도구 (성장 후 도입)

| 도구 | 기능 | 가격 |
|:---|:---|:---|
| **Repurpose.io** | 유튜브 1번 → 릴스+틱톡+쇼츠 자동 변환+배포 | $29/월 |
| **Buffer** | 채널별 포맷 변환 + 예약 게시 | 무료~$18/월 |
| **Opus Clip** | 긴 영상 → 숏폼 자동 추출 | 무료/$19/월 |

## 2026 주의사항

- 동일 콘텐츠 3개 플랫폼 그대로 올리는 크로스 포스팅은 효과 없음
- 각 플랫폼에 맞는 네이티브 콘텐츠 필요 (길이, 톤, CTA 조정)

## 실행 원칙

1. 메인 콘텐츠(릴스) 먼저 완성
2. 같은 주제로 캐러셀 구성 (슬라이드화)
3. 스레드/스토리는 핵심 인사이트 1개만 뽑아서 텍스트화
4. 각 채널 게시 후 `outputs/`에 기록
