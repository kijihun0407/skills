# 스킬: 릴스 대본 작성

> 주제 입력 → 30~40초 릴스 대본 완성본 산출.
> 전체 파이프라인: `../../../workflows/examples/reels-script-pipeline.md`

## 3-Phase 파이프라인

```
Phase 1: 리서치
  - X/인스타/스레드에서 해당 주제 최신 트렌드 확인
  - 레퍼런스 계정 (@biggie_ai/@ai_dori_/@ai_ing_) 유사 콘텐츠 확인
  - 차별화 포인트 1개 결정

Phase 2: 작성 (3버전 원칙)
  - 후킹 타입별 3가지 버전 동시 작성
    - 버전 A: 질문형 후킹 ("~하는 법 알고 있어?")
    - 버전 B: 수치형 후킹 ("N초 만에 ~")
    - 버전 C: 반전형 후킹 ("사실 ~이 아닙니다")
  - CCO가 최적 버전 선택 (CMO 전환율 관점 검수)

Phase 3: 검증
  - 품질 기준 Q1~Q6 체크
  - CTA 1개 확인
  - 캡션 + 해시태그 5개 확정
```

> **3버전 원칙**: 후킹 하나만 쓰면 AI가 가장 안전한 표현을 선택함. 3개 동시 작성 후 선택하면 최고 후킹력을 얻을 수 있다.

---

## 입력

```
주제: [AI 관련 주제 1줄]
타겟 여정 단계: 1~6 중 선택 (기본값: 1단계)
KPI 목적: 도달확장 / 리드수집 / 구매전환 (기본값: 도달확장)
```

---

## 출력 형식

```markdown
## 대본
[후킹 문장 — 첫 3초]
[핵심 메시지 — 20~30초]
[CTA — 마지막 5초]

## 캡션
[150자 이내 + 해시태그 5개]

## 검수 결과
- 글자 수: N자
- 후킹 타입: 질문형 / 반전형 / 수치형 중 선택
- CTA 개수: 1개
- 인사말 없음: ✓/✗
```

---

## 품질 기준

| ID | 기준 | 합격 조건 | 가중치 |
|:---|:---|:---|:---:|
| Q1 | 첫 문장 후킹 | 3초 내 시선 잡는 문장 | 필수 |
| Q2 | 글자 수 | 180~250자 | 필수 |
| Q3 | CTA | 정확히 1개 | 필수 |
| Q4 | 인사말 없음 | "안녕하세요/오늘은/여러분" 금지 | 필수 |
| Q5 | 전문용어 풀어쓰기 | "쉽게 말하면..." 포함 | 필수 |
| Q6 | 구체적 숫자/사례 | 추상적 표현 금지 | 권장 |
