# Directive: 주간 브리핑 SOP (운영 사업부)

> Layer 1. COO가 실행 지휘. 번아웃 없이 지속 가능한 루틴을 유지한다.

## 목표

이번 주 목표·성과·조정사항을 10분 내로 정리하고, 다음 주 우선순위를 확정한다.

## 입력값

- `../../tasks/todo.md` 현재 상태
- 이번 주 발행 콘텐츠 수
- 팔로워/수익 데이터 (있으면)
- 직장 일정 (과부하 여부)

## 실행 순서

1. `../../execution/`에 통계 수집 스크립트 있으면 실행
2. 데이터 없으면 CEO에게 요청 후 진행 (추측 금지)
3. 주간 브리핑 문서 작성
4. `../../tasks/todo.md` 업데이트
5. 저장: `outputs/{연도}/{월}/{일}/[운영]_주간브리핑.md`

## 출력 형식

```markdown
# 주간 브리핑 — {날짜}

## 이번 주 한 줄 요약
{한 줄}

## 성과

| 지표 | 목표 | 실제 | 달성률 |
|:---|:---|:---|:---|
| 릴스 발행 | 3개 | - | - |
| 팔로워 증가 | - | - | - |

## 다음 주 우선순위 TOP 3
1. ...
2. ...
3. ...

## 루틴 조정 필요 사항
...
```

## 엣지케이스

- 직장 과부하 주간 → 콘텐츠 수 줄이고 루틴 축소 버전으로 전환
- 연속 3주 목표 미달 → COO가 루틴 설계 전면 재검토 제안

## 실전 주간 루틴 (영선님 루틴 참조)

| 요일 | 업무 | AI 자동화 |
|:---|:---|:---|
| **수요일** | 레퍼런스 조사·분석 | DM으로 모아놓기, 스크롤 수집 |
| **수요일** | 콘텐츠 기획 (릴스 3~4개, 캐러셀 2개) | 자동화 프롬프트 활용 |
| **목요일** | 노션 자료·영상 소스 수집 | 정성적 부분 직접 진행 |
| **목요일 밤** | 콘텐츠 촬영 (몰아서 30분) | - |
| **토요일** | 콘텐츠 회고 (성과 분석) | 에디츠 앱 데이터 확인 |

> 루틴이 자리잡기까지 "효율적인지 모르겠다"는 의심이 들지만, 루틴이 잡히는 순간 이보다 효율적인 방법은 없다.
