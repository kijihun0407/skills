# SOP: 크론 자동화 — 24시간 자동 비서

> 크론 = 아무도 안 시켜도 정해진 시간에 자동으로 실행되는 작업.
> 신영선 강사 기준 26개 크론이 매일 24시간 가동.

---

## 3줄 패턴 — 모든 크론의 기본 구조

```bash
# 1단계: 데이터 수집
OUTPUT=$(python3 script.py)

# 2단계: Claude에게 분석
claude -p "아래 데이터 분석해라: $OUTPUT" --model claude-sonnet-4-6

# 3단계: 디스코드 알림
bash discord_notify.sh "작업 완료" success
```

> "이 3줄짜리 패턴을 26번 복사한 게 자동화 시스템."

## 신영선 강사 실패담 → 교훈

### 실패 1: 모닝 브리핑 8일간 구버전 전송
- 원인: 자동화를 만들고 검증을 안 함
- 교훈: **자동화를 만들면 반드시 1번은 직접 테스트하라**

### 실패 2: 크론 11개가 5일간 정지
- 원인: 디스코드 알림도 안 와서 몰랐음 → 광고 데이터 5일 공백
- 교훈: **시스템을 만들면 감시 시스템도 만들어라**

### 실패 3: 배포 사고
- 원인: 광고 전환 추적이 안 되는 상태로 코드 배포
- 교훈: **배포 전 결제 경로 체크 필수**

## 주간 회고 시스템 (매주 30분)

`/weekly-review` 명령 1줄 → 자동으로 생성:

| 파트 | 내용 |
|:---|:---|
| PART 1 | 김이사 요약 보고서 (핵심 인사이트) |
| PART 2 | 전주 대비 변화 |
| PART 3 | 이번 주 실행 현황 (노션 일정 기반) |
| PART 4 | KPI 대시보드 (매출/광고/콘텐츠) |
| PART 5 | 예측 + 교정 (이대로 가면 어떻게 되는가) |
| PART 6 | CEO 에너지 & 회고 |
| PART 7 | 다음 주 실행 계획 (요일별) |

### 회고 전/후 비교

| 구분 | 회고 없을 때 | 회고 있을 때 |
|:---|:---|:---|
| 현황 | "이번 주 뭐 했더라..." (기억 의존) | 노션 일정 + 실측 데이터 기반 |
| 방향 | "다음 주도 열심히" (느낌) | "KR1 D-8, 이번 주 인터뷰 없으면 미달" (숫자) |
| 소요 | - | **30분** (AI가 수집+분석, CEO는 판단만) |

## 크론 셋업 원칙

1. **3줄 패턴**만 기억: 수집 → 분석 → 알림
2. **하나씩 추가**: 한 번에 26개 아님. 필요할 때마다 1개씩
3. **감시 시스템 필수**: 크론이 멈춰도 모르면 의미 없음 → 디스코드 헬스 체크
4. **검증 후 자동화**: 만들면 반드시 1번 직접 테스트
