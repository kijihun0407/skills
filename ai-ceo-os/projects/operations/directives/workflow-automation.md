# Directive: 업무 자동화 워크플로우 SOP

> Layer 1. COO가 실행 지휘. 수동 반복 업무를 시스템으로 전환한다.

## 목표

Before(수동) → After(시스템) 전환으로 하루 4~5시간을 30분 이내로 단축한다.

---

## Before vs After 비교

### Before (수동 시절)

| 시간 | 업무 | 소요 |
|:---|:---|:---|
| 09:00 | 이메일/DM 확인 + 답변 | 30분 |
| 10:00 | 이번 주 릴스 주제 고민 | 1시간 |
| 11:00 | 대본 작성 | 2시간 |
| 14:00 | 광고 성과 확인, 엑셀 정리 | 1시간 |
| 16:00 | 내일 할 일 정리 | 30분 |

**하루 4~5시간이 "정보 수집"에 소비. 판단하기 위한 재료를 모으는 시간이 생산성을 떨어뜨림.**

### After (시스템 구축 후)

| 시간 | 업무 | 소요 |
|:---|:---|:---|
| 09:00 | AI 브리핑 확인 — 자동 요약 | 5분 |
| 10:00 | 캘린더에서 오늘 주제 확인 | 1분 |
| 11:00 | "릴스 대본 써줘" → 수정 | 15분 |
| 14:00 | "성과 분석해줘" | 3분 |
| 16:00 | todo.md 자동 업데이트 확인 | 1분 |

**총 25분. 나머지 시간은 촬영·라이브 등 대체 불가능한 작업에 집중.**

---

## 자동화 우선순위 판단 기준

```
IF 업무가 매주 반복되면 → 스킬(Skill)로 만들어라
IF 업무가 판단 없이 정해진 형식이면 → execution/ 스크립트로 만들어라
IF 업무가 여러 단계 연결이면 → workflow로 만들어라
IF 업무가 사람만 할 수 있으면 → 그것에만 집중해라
```

## 자동화 가능 업무 목록 (우선순위 순)

| 순위 | 업무 | 자동화 방법 | 예상 절감 시간 |
|:---|:---|:---|:---|
| 1 | 릴스 대본 작성 | 스킬 (`reels-script.md`) | 주 2~3시간 |
| 2 | DM 응대 | 소셜비즈 자동화 | 주 1~2시간 |
| 3 | 주간 브리핑 | COO directive | 주 30분 |
| 4 | 콘텐츠 캘린더 | 스킬 자동 생성 | 주 30분 |
| 5 | 성과 분석 | execution/ 스크립트 | 주 1시간 |

---

## 시스템 구축 3단계 여정

| 단계 | 비유 | 작업 | 결과 |
|:---|:---|:---|:---|
| **1단계** | 회사 설립 | CLAUDE.md + C-Suite 배치 | AI 회사 존재 |
| **2단계** | 직원 교육 | context/ + directives/ 채우기 | AI가 내 사업을 안다 |
| **3단계** | 실전 투입 | 스킬 실행 + 워크플로우 가동 | AI가 진짜 일한다 |

---

## 운영 원칙

1. **매일 조금씩 쌓아라** — 처음부터 다 만들면 실패. 매주 1개씩 추가.
2. **CLAUDE.md가 모든 것을 결정한다** — 1층이 기울면 10층도 기울어짐.
3. **시스템이 사람을 이긴다** — 사람은 까먹지만 시스템은 6개월 전 데이터도 기억.

---

## 자동화 가능 여부 판단

```
반복 업무 발생
→ "이 업무를 매주 하나?" YES → 스킬화
→ "판단 없이 정해진 형식인가?" YES → 스크립트화
→ "사람만 할 수 있나?" YES → 그것에만 집중
```
