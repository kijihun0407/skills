# 운영 루틴 · 스케줄 기준 — 운영 사업부

## 주간 콘텐츠 루틴

| 요일 | 작업 | 예상 소요 |
|:---|:---|:---:|
| 월 | 릴스 1 발행 + 주간 브리핑 | 30분 |
| 화 | 다음 주 주제 리서치 | 30분 |
| 수 | 릴스 2 발행 + 대본 초안 1개 | 20분 |
| 목 | 대본 검수 · 수정 | 20분 |
| 금 | 릴스 3 발행 | 10분 |
| 토 | 카드뉴스 기획 or 휴식 | 선택 |
| 일 | 다음 주 대본 3개 완성 | 1시간 |

> 직장 병행 기준 — 하루 최대 1시간 초과 금지

## 세션 관리 원칙

- 작업 시작: `tasks/todo.md` 확인
- 작업 종료: `tasks/session_handoff.md` 업데이트
- 실수 발생: `tasks/lessons.md` 즉시 기록

## 번아웃 방지 규칙

- 주간 콘텐츠 목표: 릴스 3개 (이 이상은 욕심)
- 루틴 2회 연속 붕괴 시 → COO 소집해서 원인 파악
- 컨디션 안 좋은 날: 릴스 1개로 축소 허용
